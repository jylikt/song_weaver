ТЗ: загрузка xcodec (m-a-p/xcodec_mini_infer) в gpu-worker
Цель
Обеспечить успешную загрузку аудио-кодека из каталога YUE_CODEC_PATH (локальный снимок xcodec_mini_infer) в процессе load-model, без ошибок вида unexpected keyword argument 'generator', Repo id must be in the form..., отсутствующих модулей (dac и т.д.).

Нормативный образец
Приложение А: файл infer.py из репозитория YuE (путь и версия — как у поставщика). Поведение загрузчика в artifacts/gpu-worker должно семантически совпадать с тем, как в этом файле:

настраивается sys.path;
читается config.yaml из final_ckpt/;
инстанцируется класс генератора/кодека (eval(...)(**...) и вложенный конфиг, а не «весь YAML целиком» в конструктор);
подгружаются веса из ckpt_*.pth;
резолвятся локальные пути к вспомогательным артефактам (в т.ч. semantic_ckpts), чтобы строки не уходили в Hugging Face Hub как repo_id.
Текущая проблема (кратко)
В конструктор модели передаётся неверный набор kwargs (верхний уровень YAML с полем generator вместо generator.config + правильного класса по generator.name).
Относительные пути из конфига (например ./xcodec_mini_infer/...) не приводятся к путям от YUE_CODEC_PATH.
В sys.path недостаточно путей по сравнению с infer.py (как минимум корень кодека и каталог descriptaudiocodec).
Требования к доработке artifacts/gpu-worker
Инициализация модели: повторить логику из Приложения А для чтения YAML и создания экземпляра (имя класса + kwargs из вложенного конфига), без fallback SoundStream() без аргументов, если это приводит к несовместимости с чекпоинтом.
Загрузка весов: как в Приложении А — извлечь state_dict из .pth (ключи model / state_dict / codec_model и т.д. — по факту структуры чекпоинта в образце).
Пути: рекурсивно (или точечно по известным полям конфига) заменить/нормализовать префиксы вроде ./xcodec_mini_infer/ на абсолютный путь внутри codec_dir (YUE_CODEC_PATH).
sys.path: добавить те же корни, что в Приложении А (относительно YUE_CODEC_PATH), чтобы работали импорты dac, models.soundstream_hubert_new, и т.д.
Логирование: при ошибке логировать исходное исключение и этап (парсинг конфига / eval / load_state_dict / Hub), не маскируя первопричину финальным фолбэком xcodec2-pkg.
Критерии приёмки
При валидных YUE_CODEC_PATH, весах и зависимостях venv POST /load-model завершается с codec_loaded=True (или эквивалент в логах: успешная строка загрузки xcodec repo).
Нет обращения Hub с строкой ./xcodec_mini_infer/... как к repo_id.
Токенизатор YuE по-прежнему загружается (отдельно: зафиксировать protobuf>=4.21 в requirements.txt gpu-worker, чтобы pip не откатывал его при установке аудио-зависимостей).
